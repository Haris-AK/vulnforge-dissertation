# CM3203 guide notes for student help

Use these points only when they are relevant to the question or draft shown.

These notes are a **fallback summary**, not a substitute for checking the live guide when the answer depends on current wording.

## Module and deliverables

- CM3203 is the **one semester individual project**.
- The work is towards two assessments:
  - **Initial plan** worth **5%**
  - **Final report** worth **95%**
- The guide gives the usual timing:
  - initial plan by the **beginning of spring week 2**
  - final report by the **end of spring week 12**
- The **exact submission deadlines** are shown in the student's **PATS project details**.
- The guide says the management and execution of the project are the **student's responsibility**, with advice from the supervisor.

## Initial plan essentials

- The initial plan is a **single PDF** submitted via PATS.
- The main body has a maximum of **1,500 words**.
- The expected core sections are:
  - project description
  - aims and objectives
  - feasibility
  - work plan
- The structure should follow the guide closely.
- Significant project changes should be agreed with the supervisor.

## Final report essentials

- The final report has a maximum of **12,000 words for the main body**.
- The word count is a **maximum, not a target**.
- The count excludes title page, abstract, acknowledgements, contents, references, figures, tables, appendices, and separately submitted supplementary/support material.
- The report should give enough information that a **competent computer scientist** can understand and reproduce the work.
- The report should cover the problem/background, solution, and evaluation.
- Source code should be submitted where applicable.

## Submission and support files

- PATS separates **document files** from **support files**.
- Document files are the PDF report files.
- Support files are supplementary evidence such as code, selected data, configuration files, results, or other artefacts.
- Large raw datasets, bulky binaries, videos, trained models, or other very large files may need discussion with the supervisor rather than automatic upload.
- A README can help when the support files need explanation.
- Files count as submitted only after the student completes the submission process.

## Project execution and supervision

- Students are expected to organise and take responsibility for progress.
- Students should take advantage of supervisor advice.
- If major project directions or deliverables change, this should be discussed with the supervisor.
- If a supervision/process issue genuinely blocks progress, the guide may point students to the coordinator.

## AI and tool use

AI tools are permitted and should be treated like other project tools.

Where their use is material to the project, students should:
- explain what was used and for what purpose
- justify why that use was appropriate
- describe important limitations and how outputs were checked or validated
- acknowledge and reference the tool appropriately

Students should not cite an AI tool instead of the underlying scholarly or technical source when the underlying source is what supports the claim.

Students should not upload confidential, personal, sensitive, restricted, or otherwise protected material to external tools unless that is permitted by the relevant ethics, legal, licensing, and data-protection requirements.

## Feedback posture

When using these notes:
- mention only the guide points that matter to the student's question
- prefer brief, actionable reminders over long compliance lectures
- separate current guide facts from your own practical interpretation
- never imply that guide compliance alone determines quality
